# 2026-06-04 작업 변경 기록

## 요약

2026-06-04 작업은 협력사 자료 요청을 실제 업무 큐로 처리하는 흐름을 정리하고, 협력사 상세 화면과 제품별 공급망 점검 화면의 UI 구조를 함께 정돈한 변경이다.

## 자료 요청 업무 보드

- `/supply-chain/request-map`을 공급망 지도 중심 화면에서 자료 요청 업무 보드로 개편했다.
- 상단 KPI, 좌측 업무 큐, 중앙 요청 상세, 우측 다음 조치 패널 구조로 재구성했다.
- 상태 색상을 통일했다.
  - 기한 초과: 빨강
  - 검토 대기: 초록
  - 만료 임박: 주황
  - 입력 중: 파랑
  - 검토 완료: 회색
  - 보완 요청: 보라
  - 반려: 검정
- 좌측 업무 큐 순서를 `기한 초과 -> 검토 대기 -> 보완 요청 -> 반려 -> 만료 임박 -> 입력 중` 흐름으로 정리했다.
- 기본 업무 큐에서는 검토 완료 항목을 숨기고, `완료` 필터에서 최근 완료 항목을 확인하도록 했다.
- 업무 큐 필터 버튼을 `전체 / 초과 / 임박 / 입력중 / 대기 / 완료` 한 줄로 정렬했다.
- 검토 대기 항목 클릭 시 팝업이 뜨고, 팝업에서 제출 현황 페이지 이동 또는 즉시 검토 완료 처리가 가능하게 했다.
- 검토 대기 팝업의 이동 경로 안내 문구는 제거했다.
- 영향받는 제품 / 공급망 경로 영역에서 협력사별 상태 색상을 표시하게 했다.

## 제출 자료 검토 연동

- `/submission-review`의 우측 하단 결정 버튼이 request-map 상태와 동기화되도록 연결했다.
- `승인` 또는 `HITL 요청` 클릭 시 request-map의 검토 대기 항목이 검토 완료 회색 상태로 바뀐다.
- `보완 요청` 클릭 시 request-map의 해당 항목이 보완 요청 보라색 상태로 바뀐다.
- `반려` 클릭 시 request-map의 해당 항목이 반려 검정색 상태로 바뀐다.
- 페이지 간 상태 전달은 URL 파라미터와 브라우저 저장 상태를 함께 사용하도록 했다.

## 공급망 / 네비게이션

- 네비게이션 라벨을 변경했다.
  - `제품별 공급망 맵` -> `제품별 DPP 공급망 점검`
  - `입력 요청 맵` -> `자료 요청 업무 보드`
- `/supply-chain/product-map` 화면 제목도 제품별 DPP 공급망 점검 톤으로 맞췄다.

## 협력사 상세 화면 리팩터링

- 협력사 상세 하위 화면을 `components/supplier-pages` 구성으로 분리했다.
- 일반 정보, 원산지, ESG, FEOC, AI 검증, 교육 화면의 반복 UI를 공통 컴포넌트와 섹션 컴포넌트로 나눴다.
- `supplier-detail-scope` 스타일을 추가해 협력사 상세 영역의 글자 크기, 카드 배경, 여백 톤을 조정했다.
- `NavLink`에 하위 메뉴 exact 매칭 옵션을 추가해 중첩 경로에서 활성 메뉴가 더 정확하게 표시되도록 했다.
