## 현재 상황 요약

KIRA 대시보드는 밝은 운영 관제 시스템 톤으로 개편 중이다. 사용자는 "예쁜 카드 모음"보다 실제 운영자가 쓰는 엔터프라이즈 SaaS 콘솔 같은 화면을 원한다.

핵심 방향:

- 정보량은 유지한다.
- 기능/데이터/로직은 건드리지 않는다.
- 카드와 박스가 너무 많아 답답해 보이는 문제를 줄인다.
- 메인 정보, 보조 정보, 액션 정보의 위계를 더 분명히 한다.
- 기존 구조를 전면 재작성하기보다 현재 구조를 살리며 레이아웃과 스타일을 개선한다.

## 사용자가 마음에 안 들어 한 지점

`materials` 화면에서 왼쪽 자재 목록과 오른쪽 상세/보조 패널의 하단 단차가 거슬린다고 했다.

한 번 시도했던 수정:

- 왼쪽 목록에 `min-h-[660px]`
- 목록 아래 빈 영역을 같은 패널 배경으로 채움
- 오른쪽 영역을 `grid-rows-[1fr_auto]`
- 메인 상세를 `flex`로 늘림

결과:

- 사용자가 "더 구리다"고 판단했다.
- 그래서 이 높이 고정 패치는 되돌렸다.

현재 상태:

- 리디자인 자체는 남아 있다.
- 단차 맞춤용 `min-h/flex/grid-row` 패치만 제거된 상태다.

다음 방에서 단차 문제를 다시 건드린다면, 억지 높이 맞춤보다 다음 방향이 낫다.

- 왼쪽 목록과 오른쪽 상세를 같은 박스 높이로 맞추려 하지 말고, 아예 레이아웃 구조를 바꾼다.
- 예: 왼쪽 목록을 sticky sidebar처럼 만들고 오른쪽 상세만 문서처럼 흐르게 둔다.
- 예: 보조 패널을 오른쪽 상세 안쪽 하단이 아니라 별도 우측 rail 또는 접이식 영역으로 둔다.
- 예: 선택 자재 상세 안에 증빙/PO를 탭 또는 segmented control로 흡수해 하단 두 카드 단차를 없앤다.
- 단순히 빈 공간을 채우는 방식은 답답해 보이므로 피한다.

## 디자인 개선 원칙

1. 박스 수를 줄인다.
   - 카드 안에 작은 카드가 반복되는 구조를 피한다.
   - 메타 정보는 가벼운 label-value row나 inline meta로 표시한다.

2. 메인/보조 위계를 분명히 한다.
   - 현재 선택된 대상의 상세가 가장 커야 한다.
   - 증빙, PO, 참고 기준은 더 낮은 톤이어야 한다.

3. 여백은 "큰 패딩"이 아니라 "구획 수 감소"로 만든다.
   - 칸막이를 많이 만들지 않는다.
   - 구분선은 꼭 필요한 곳에만 쓴다.

4. 운영 화면답게 만든다.
   - 어디를 먼저 봐야 하는지 바로 보여야 한다.
   - 상태 색상은 작고 명확하게 쓴다.
   - 액션 버튼은 흐릿하지 않게, 그러나 과하게 크지 않게 둔다.

5. 단차를 억지로 맞추지 않는다.
   - 고정 높이로 빈 공간을 채우면 더 답답해질 수 있다.
   - 정보 구조를 바꾸거나 보조 정보의 위치를 재배치하는 방식으로 해결한다.

## 다음 작업 추천

우선 `materials` 페이지에서 다음 중 하나를 선택해 다시 잡는 것이 좋다.

### 방향 A: 상세 중심 문서형

- 왼쪽 목록은 320~360px sticky list로 둔다.
- 오른쪽은 단일 큰 상세 문서처럼 흐르게 만든다.
- 증빙/PO를 메인 상세 아래의 두 카드가 아니라 상세 내부 섹션으로 흡수한다.
- 장점: 단차 스트레스가 가장 적다.

### 방향 B: 탭 흡수형

- 오른쪽 상세 안에 `조성 / 증빙 / PO` 탭 또는 segmented control을 둔다.
- 하단 보조 카드 두 개를 제거한다.
- 장점: 한 화면 높이가 안정되고 덜 조각나 보인다.

### 방향 C: 우측 보조 rail

- 가운데는 물질 조성 메인.
- 오른쪽 narrow rail에 증빙과 PO 요약을 쌓는다.
- 장점: 운영 콘솔 느낌이 강하지만, 화면 폭을 많이 쓴다.

현재 사용자 반응을 보면 방향 B가 가장 무난하다. "긴 파일 누르면 더 길어짐" 문제도 탭 흡수형이면 줄어든다.

```powershell
$workspace = Resolve-Path .
$target = Resolve-Path .next -ErrorAction SilentlyContinue

  Get-Process -Name node -ErrorAction SilentlyContinue |

    Stop-Process -Force
```
